# 个人知识库系统操作指南

## 一、项目初始化与部署

### 1. GitHub仓库创建与配置

1. **创建仓库**
   - 登录GitHub，点击「New repository」创建新仓库
   - 仓库名称建议使用：`username.github.io`（username是你的GitHub用户名）
   - 选择「Public」公开仓库
   - 勾选「Initialize this repository with a README」

2. **开启GitHub Pages**
   - 进入仓库「Settings」→「Pages」
   - 选择「Branch: gh-pages」，点击「Save」
   - 稍等片刻，你的网站将可以通过 `https://username.github.io` 访问

3. **配置GitHub Actions权限**
   - 进入仓库「Settings」→「Actions」→「General」
   - 在「Workflow permissions」中选择「Read and write permissions」
   - 勾选「Allow GitHub Actions to create and approve pull requests」
   - 点击「Save」

### 2. 本地项目初始化

1. **克隆仓库到本地**
   ```bash
   git clone https://github.com/username/username.github.io.git
   cd username.github.io
   ```

2. **复制项目文件**
   - 将所有项目文件（_posts、_layouts、assets、_config.yml等）复制到仓库根目录

3. **更新配置文件**
   - 编辑 `_config.yml` 文件，更新以下信息：
     ```yaml
     name: "我的个人知识库"        # 网站名称
     description: "技术学习和问题解决的知识库"  # 网站描述
     url: "https://username.github.io"  # 替换为你的GitHub Pages URL
     ```

4. **提交并推送代码**
   ```bash
   git add .
   git commit -m "初始化个人知识库系统"


   git remote add origin https://github.com/ribatsua/cp25.github.io.git
   ```

5. **等待自动部署**
   - GitHub Actions将自动执行构建和部署流程
   - 部署完成后，访问你的GitHub Pages URL即可看到网站

## 二、日常使用流程

### 1. 编写Markdown文章

1. **创建新文章**
   - 在 `_posts` 目录下创建新文件，命名格式：`YYYY-MM-DD-文章标题.md`
   - 例如：`2025-12-23-markdown语法入门.md`

2. **编写文章内容**
   - 文章开头添加Front Matter元数据（必须）
   - 然后编写Markdown正文内容
   - 示例：
     ```markdown
     ---
     title: "Markdown语法入门"
     date: 2025-12-23
     tags: [Markdown, 语法, 教程]
     ---

     # Markdown语法入门

     欢迎阅读本文...
     
     <!-- more -->

     文章的完整内容...
     ```

### 2. 提交与更新网站

1. **提交Markdown文件**
   ```bash
   git add _posts/2025-12-23-markdown语法入门.md
   git commit -m "添加Markdown语法入门文章"
   git push origin main
   ```

2. **自动构建与部署**
   - GitHub Actions会自动检测到代码变更
   - 自动执行构建流程，将Markdown转换为HTML
   - 自动部署到GitHub Pages
   - 部署时间通常为1-2分钟

3. **查看更新后的网站**
   - 访问你的GitHub Pages URL
   - 刷新页面即可看到最新的文章内容

## 三、功能使用说明

### 1. 主题切换

- **手动切换**：点击右上角的月亮/太阳图标
- **自动检测**：系统会根据你的操作系统主题偏好自动切换
- **快捷键**：`Ctrl/Cmd + T` 快速切换主题

### 2. 内容搜索

- **打开搜索**：点击右上角的搜索图标，或使用快捷键 `Ctrl/Cmd + K`
- **搜索内容**：在搜索框中输入关键词，系统会实时筛选匹配的文章
- **搜索范围**：文章标题、内容摘要和标签
- **关闭搜索**：点击关闭按钮、点击遮罩层或按 `ESC` 键

### 3. 标签筛选

- **标签云**：首页顶部显示所有文章标签
- **点击筛选**：点击任意标签，系统会筛选出包含该标签的所有文章
- **再次点击**：点击已激活的标签可取消筛选
- **标签数量**：每个标签后显示包含该标签的文章数量

### 4. 移动端适配

- **响应式设计**：网站会根据屏幕尺寸自动调整布局
- **移动端菜单**：点击左上角的菜单图标打开导航菜单
- **触摸优化**：所有交互元素都经过触摸优化，适合手机和平板设备

### 5. 代码高亮

- **自动高亮**：所有代码块会自动应用语法高亮
- **支持语言**：支持JavaScript、Python、HTML、CSS等多种编程语言
- **主题匹配**：代码高亮主题会根据网站主题自动切换

## 四、Markdown写作规范

### 1. Front Matter元数据

每篇文章必须包含以下元数据：

```yaml
---
title: "文章标题"       # 文章标题
date: 2025-12-23        # 发布日期
tags: [标签1, 标签2]     # 文章标签，用于分类筛选
---
```

### 2. 内容结构

- **标题层级**：使用 `#` 到 `######` 定义标题层级
- **段落分隔**：段落之间使用空行分隔
- **摘要分隔**：使用 `<!-- more -->` 分隔文章摘要和完整内容
- **代码块**：使用三个反引号包裹代码块，并指定语言

### 3. 推荐写作技巧

- **使用标题**：清晰的标题结构有助于阅读和理解
- **添加标签**：合理使用标签便于文章分类和筛选
- **代码示例**：技术文章建议添加完整的代码示例
- **图片说明**：为图片添加适当的描述和标题
- **保持简洁**：内容简洁明了，重点突出

## 五、常见问题与解决方案

### 1. 网站无法访问

- **检查GitHub Pages配置**：确认分支选择正确（gh-pages）
- **检查GitHub Actions状态**：进入仓库「Actions」查看构建是否成功
- **等待部署完成**：新部署的网站可能需要几分钟才能生效

### 2. 文章没有显示在首页

- **检查文件名格式**：确保文件名格式为 `YYYY-MM-DD-文章标题.md`
- **检查Front Matter**：确保文章包含正确的元数据
- **检查发布日期**：发布日期不能晚于当前日期

### 3. 代码高亮不生效

- **检查代码块格式**：确保代码块使用三个反引号包裹
- **指定语言**：在反引号后指定编程语言，例如 ```javascript
- **浏览器缓存**：尝试清除浏览器缓存后重新加载页面

### 4. 主题切换不生效

- **检查浏览器支持**：确保使用现代浏览器（Chrome、Firefox、Safari等）
- **清除缓存**：清除浏览器缓存后重新加载页面
- **检查JavaScript**：确保浏览器未禁用JavaScript

### 5. 搜索功能不工作

- **检查文章数量**：确保有足够的文章可供搜索
- **检查搜索关键词**：尝试使用更宽泛的关键词
- **检查JavaScript**：确保浏览器未禁用JavaScript

## 六、高级功能与扩展

### 1. 自定义样式

- **修改CSS变量**：在 `assets/css/main.css` 中修改 `:root` 下的CSS变量
- **添加自定义样式**：在 `main.css` 文件末尾添加自定义样式规则
- **响应式调整**：修改媒体查询规则适配特定设备

### 2. 添加新页面

- **创建新页面**：在根目录创建新的HTML文件（如 `about.html`）
- **使用模板**：在页面顶部添加 `--- layout: default ---` 使用默认模板
- **更新导航**：在 `_config.yml` 中的 `navigation` 部分添加新页面链接

### 3. 自定义域名

1. **购买域名**：从域名注册商（如阿里云、腾讯云等）购买域名
2. **配置DNS**：添加CNAME记录指向 `username.github.io`
3. **GitHub配置**：进入仓库「Settings」→「Pages」→「Custom domain」，输入你的域名

## 七、备份与恢复

### 1. 数据备份

- **GitHub自动备份**：所有内容都保存在GitHub仓库中，自动进行版本控制
- **本地备份**：定期将仓库克隆到本地作为额外备份
- **导出Markdown**：可以将 `_posts` 目录下的所有Markdown文件导出保存

### 2. 数据恢复

- **版本回退**：使用Git命令回退到指定版本
  ```bash
  git log  # 查看提交历史
  git checkout <commit-id>  # 切换到指定版本
  ```
- **重新部署**：恢复后重新推送代码到GitHub，自动触发部署
