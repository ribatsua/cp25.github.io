# 创建新文章指南

## 一、文章命名规则

在Jekyll中，文章文件需要遵循特定的命名规则，才能被正确识别和生成页面：

### 命名格式
```
YYYY-MM-DD-文章标题.md
```

### 示例
```
2025-12-23-Markdown语法入门与实践.md
2025-12-24-JavaScript基础语法.md
```

### 注意事项
1. 日期部分必须是**年-月-日**格式（如：2025-12-23）
2. 标题部分使用**英文连字符**（-）分隔单词
3. 文件名必须以**.md**结尾（Markdown格式）

## 二、文章存放位置

所有文章文件必须存放在项目根目录下的**_posts**目录中：

```
个人知识库/
└── _posts/
    └── YYYY-MM-DD-文章标题.md
```

## 三、文章Front Matter格式

每篇文章的开头必须包含Front Matter（前置元数据），用于定义文章的属性。

### 基本格式
```yaml
---
title: "文章标题"
date: YYYY-MM-DD
tags: [标签1, 标签2, 标签3]
---
```

### 示例
```yaml
---
title: "Markdown语法入门与实践"
date: 2025-12-23
tags: [Markdown, 语法, 教程]
---

# 文章内容开始...
```

### 字段说明
- **title**：文章的显示标题（使用英文双引号包裹）
- **date**：文章的发布日期（格式：YYYY-MM-DD）
- **tags**：文章的标签（使用方括号包裹，多个标签用逗号分隔）

## 四、创建新文章的两种方式

### 方式一：本地创建（推荐）

1. **打开本地项目**：使用VS Code等编辑器打开本地知识库项目
2. **创建新文件**：在`_posts`目录下创建新的Markdown文件
3. **命名文件**：按照命名规则命名文件（如：2025-12-24-JavaScript基础.md）
4. **编写Front Matter**：在文件开头添加Front Matter
5. **编写内容**：使用Markdown语法编写文章内容
6. **提交并推送**：
   ```bash
   git add .
   git commit -m "添加新文章：JavaScript基础"
   git push origin main
   ```

### 方式二：GitHub网页创建

1. **登录GitHub**：进入仓库页面（https://github.com/ribatsua/cp25.github.io）
2. **进入_posts目录**：点击"_posts"文件夹
3. **创建新文件**：点击右上角的"Add file" → "Create new file"
4. **命名文件**：在文件名输入框中按照规则命名
5. **编写内容**：在编辑框中添加Front Matter和文章内容
6. **提交更改**：
   - 填写提交信息（如："添加新文章：JavaScript基础"）
   - 选择"Commit directly to the main branch"
   - 点击"Commit new file"

## 五、自动部署流程

无论使用哪种方式创建文章，提交后都会自动触发部署流程：

1. **提交触发**：向`main`分支提交更改后，GitHub Actions自动启动
2. **构建生成**：Jekyll将Markdown文件转换为静态HTML页面
3. **部署发布**：构建完成后自动部署到`gh-pages`分支
4. **访问更新**：等待1-2分钟后，访问网站即可看到新文章

## 六、Markdown语法提示

### 标题
```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 列表
```markdown
- 无序列表项1
- 无序列表项2
  - 子列表项

1. 有序列表项1
2. 有序列表项2
```

### 代码
```markdown
行内代码：`console.log("Hello")`

代码块：
```javascript
function hello() {
    console.log("Hello, World!");
}
```
```python
def hello():
    print("Hello, World!")
```
```html
<div class="container">
    <h1>标题</h1>
</div>
```
```css
.container {
    max-width: 1200px;
    margin: 0 auto;
}
```

### 链接和图片
```markdown
[链接文本](链接地址 "链接标题")

![图片描述](图片地址 "图片标题")
```

### 表格
```markdown
| 表头1 | 表头2 | 表头3 |
|------|------|------|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 引用
```markdown
> 这是一段引用文本。
> 
> > 这是嵌套引用。
```

### 强调
```markdown
*斜体文本*
**粗体文本**
***粗斜体文本***
```

## 七、本地预览

如果需要在本地预览文章效果，可以使用以下方法：

### 方法一：使用GitHub Desktop（推荐）
1. 安装并打开GitHub Desktop
2. 打开知识库项目
3. 点击"Preview"按钮预览网站

### 方法二：使用命令行
1. 确保已安装Ruby和Jekyll
2. 在项目根目录运行：
   ```bash
   bundle install
   bundle exec jekyll serve
   ```
3. 在浏览器中访问：http://localhost:4000

## 八、注意事项

1. **文章日期**：日期会影响文章在首页和归档中的显示顺序（最新的在前）
2. **标签管理**：标签应尽量统一，避免重复或拼写错误
3. **内容长度**：建议使用`<!-- more -->`分隔符来控制首页显示的摘要长度
4. **图片引用**：建议使用外部图片链接（如GitHub仓库、图床等）
5. **代码规范**：保持良好的Markdown格式，提高可读性

## 九、常见问题

### 1. 文章不显示在首页？
- 检查文件名是否符合日期格式
- 检查是否存放在`_posts`目录中
- 检查Front Matter是否正确填写

### 2. 部署后文章没有更新？
- 等待1-2分钟后刷新页面
- 检查GitHub Actions是否构建成功
- 检查文章的日期是否正确

### 3. 图片显示不出来？
- 检查图片链接是否正确
- 检查图片是否有访问权限
- 考虑使用GitHub仓库中的图片（相对路径）
